## IMAGE SEO OPTIMIZATION

### File Naming Convention

**Estructura de nombres:**
```
[marca]-[producto/categoria]-[variante]-[descriptor]-[dimension].extension

✗ IMG_12345.jpg (evitar)
✗ foto-nueva-final-v2.jpg (evitar)
```

### Alt Text Guidelines

**Estructura de alt text:**
```
[Acción/Estado] + [Objeto Principal] + [Ubicación/Contexto] + [Marca si relevante]

Ejemplos:
✓ "Botella Viña Santa Cruz Carmenere Gran Reserva 2020 Valle del Maipo fondo blanco"
✓ "Grupo de turistas caminando entre hileras de vides centenarias Viña Santa Cruz"
✓ "Copa vino tinto con viñedos Valle del Maipo y montañas Andes al fondo"
✓ "Barricas de roble francés en cava subterránea iluminada bodega Santa Cruz"
✓ "Teleférico panorámico ascendiendo sobre viñedos con vista cordillera"

✗ "imagen de vino" (muy genérico)
✗ "carmenere" (insuficiente contexto)
✗ "Compra ahora el mejor vino" (keyword stuffing)
```

### Image Optimization Specs

**Formatos recomendados:**
- WebP para web moderna (savings 25-35% vs JPEG)
- JPEG como fallback
- PNG solo para logos/iconos con transparencia
- SVG para iconos vectoriales

**Dimensiones estándar:**
- Hero images: 1920x1080px (16:9)
- Product images: 800x1200px (2:3) o 1000x1000px (1:1)
- Thumbnails: 400x400px
- Blog featured: 1200x630px (OG image ratio)
- Mobile hero: 750x1334px

**Compresión:**
- Tamaño máximo por imagen: 200KB
- Herramientas: TinyPNG, ImageOptim, Squoosh
- Quality setting: 80-85 para JPEG

**Lazy Loading:**
```html
<img src="carmenere-gran-reserva-2020.jpg"
     alt="Botella Viña Santa Cruz Carmenere Gran Reserva 2020"
     loading="lazy"
     width="800"
     height="1200" />
```


## PERFORMANCE SEO REQUIREMENTS

### Core Web Vitals Targets

**LCP (Largest Contentful Paint):**
- Target: ≤ 2.5 segundos
- Optimizations:
  - Hero images en WebP <200KB
  - Lazy load para imágenes below fold
  - CDN para assets estáticos
  - Preload de imágenes hero

**FID (First Input Delay):**
- Target: ≤ 100 milisegundos
- Optimizations:
  - Defer JavaScript non-critical
  - Minimize JavaScript execution time
  - Code splitting para reducir bundle size

**CLS (Cumulative Layout Shift):**
- Target: ≤ 0.1
- Optimizations:
  - Especificar width/height en todas las imágenes
  - Reservar espacio para ads/embeds
  - Evitar insertar contenido dinámico above fold
  - Font display: swap con fallback similar

### Mobile-First Optimization

**Responsive Breakpoints:**
```css
/* Mobile first approach */
/* Base styles: Mobile (320px+) */

@media (min-width: 768px) {
  /* Tablet */
}

@media (min-width: 1024px) {
  /* Desktop */
}

@media (min-width: 1440px) {
  /* Large desktop */
}
```

**Touch Targets:**
- Mínimo 44x44px para botones/links
- Espaciado mínimo 8px entre elementos clickeables
- CTAs principales: 48x48px mínimo

**Font Sizes:**
- Body text mobile: 16px mínimo (no 14px)
- Headings mobile: 24px (H1), 20px (H2), 18px (H3)
- Line height: 1.5-1.6 para legibilidad
